# あとがき

本リポジトリで扱っている技術領域 — devcontainer のネットワーク隔離やプロキシ経由の通信制御 — は、見渡してみると、大企業の情報システム部門 (情シス) が組織内の従業員端末に対して敷いてきた制御と同じ系統のものだ。事例を探してもまず大企業向けの文脈が出てくる。本リポジトリは、それと同じ構造を個人開発者の手元の devcontainer 1 つに対して組み直している。

個人の手元でなぜ同じ制御が要るかというと、AI エージェントが任意コード実行能力を持って開発者の手元で動くようになったからだ。秘匿情報の持ち出しや認証情報の意図しない送信といった、情シスが対処してきた問題が、AI エージェントの普及で個人開発者のところまで降りてきた形だと思う。

ただ、需要があっても組み立てる手段がなければ実現できない。もう一つ面白いのは、手段の側も同時に揃っていたことだ。本リポジトリで使っている材料は mitmproxy や Caddy、Docker Compose といった OSS ばかりで、個人の PC 上で組み立てられる。情シスが専用の装置や設備で実現してきたことと同じ構造が、手元のツールの組み合わせで成り立ってしまう。

需要と手段のどちらか一方でも欠けていたら、こういう構成を個人の手元で試すこと自体ができなかった。両方が同時に揃ったから個人の試行として成り立っている。面白い時代になったなと思う。
